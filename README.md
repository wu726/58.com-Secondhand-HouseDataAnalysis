# 武汉二手房数据分析与房价预测

从 **58同城** 爬取武汉市多区域二手房数据，经过清洗、特征工程、可视化分析，使用 **LightGBM** 构建房价预测模型的完整项目。

## 数据采集

`58爬虫脚本.py` 基于 `requests` + `BeautifulSoup` 爬取 58同城武汉二手房信息：
- 支持按 **区域**（修改 URL 中的拼音段）和 **装修类型**（URL 参数 `j1` 毛坯 / `j2` 简装 / `j4` 精装 / `j5` 豪装）爬取
- 请求头随机化（`fake_useragent`）、Cookie 维持登录态、随机延迟模拟人类行为
- 爬取字段：小区名称、户型、面积、朝向、楼层、建造年份、总价、单价、核心卖点

原始数据分别存入 `data/` 下6个行政区的 CSV 文件。

## 数据流水线（处理流程）

| 步骤 | 脚本 | 说明 |
|------|------|------|
| **1** | `1.处理合并爬虫数据.py` | 遍历各区域文件夹，按 4 类装修类型合并原始爬虫数据，添加区域/装修类型标签 |
| **2** | `2.统一格式.py` | 拆分户型（提取 X室X厅）和楼层（提取总楼层），统一字段命名和顺序 |
| **3** | `3.标准化格式.py` | 拆分室/厅/卫为独立数值列，提取楼层类型（高/中/低），移除冗余字段，生成 6 个区域的标准格式文件 |
| **4** | `4.合并所有区域数据.py` | 将 6 个区域标准格式数据纵向合并为全量数据集 |
| **5** | `5.清洗数据.py` | 剔除空值/未知值行、删除完全重复行、重置索引 |
| **6** | `6.划分数据集.py` | 基于区域/朝向/楼层分层抽样 7:3 划分训练/测试集，生成特征：**房龄、卫室比、厅室比、房间总数、功能区密度** |
| **7** | `7.类别数据编码映射.py` | 类别特征编码：朝向（固定映射）、户型（按字母序）、区域（按总价中位数动态间隔编码）、装修类型、楼层；生成 JSON 映射表供测试集复用 |
| **8** | `8.归一标准化.py` | 连续数值特征（面积、卫室比、厅室比、功能区密度）做 MinMax 归一化（0-1），保存 Scaler 参数 |
| **9** | `9.lightGBM模型预测房价.py` | LightGBM 回归（Poisson 损失），早停机制，输出特征重要性、预测结果表、模型文件 |

### 弃用脚本（`弃用/`）

| 脚本 | 说明 |
|------|------|
| `剔除缺失值.py` | 旧版按区域分别剔除缺失值（后被整合进步骤5） |
| `剔除重复值.py` | 旧版单独去重脚本（后被整合进步骤5） |
| `lightGBM模型预测房价.py` | 旧版模型（StandardScaler + 逆变换，后被 MinMax + 直接预测取代） |

## 技术栈

- **爬虫**: requests, BeautifulSoup, fake_useragent

## 运行说明
